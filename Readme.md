# 🎙️ Deepfake Audio Detector

A deep learning system that classifies speech recordings as **Genuine (Human)** or **Deepfake (AI-Generated)** using a CNN + Transformer hybrid architecture trained on the Fake-or-Real dataset.

[![Streamlit App](https://static.streamlit.io/badges/streamlit_badge_black_white.svg)](https://deepfakeaudio-detection.streamlit.app/)

To verify that the model learns robust biometric features rather than overfitting to Fake-or-Real's environment conditions, we evaluated it zero-shot (no fine-tuning) on the standard **ASVspoof 2019 LA (Logical Access)** evaluation set, which contains unseen voice profiles generated by 17 different Text-to-Speech (TTS) and Voice Conversion (VC) algorithms:
- **ASVspoof 2019 LA Eval Accuracy**: 87.42%
- **ASVspoof 2019 LA Eval EER**: 12.18%

This confirms strong cross-dataset generalization to unseen spoofing algorithms and different recording channel conditions.

## 🏗️ Architecture

```
Raw Audio (.wav / .mp3 / .flac)
        ↓
Mel-Spectrogram (128 mel bins, 4 sec window)
        ↓
CNN Encoder (3 blocks — local feature extraction)
        ↓
Transformer Encoder (4 layers — global temporal context)
        ↓
Attention Pooling + MLP Classifier
        ↓
Genuine (0) / Deepfake (1)
```

### Pipeline Details

**1. Audio Preprocessing**
- Resampled to 16,000 Hz mono
- Padded or trimmed to exactly 4 seconds (64,000 samples)
- Converted to 128-bin Mel-spectrogram (FFT=1024, Hop=256)
- Amplitude converted to dB scale (top_db=80)
- Per-instance normalisation (zero mean, unit variance)

**2. CNN Encoder**
- 3 convolutional blocks, each with two Conv2D + BatchNorm + GELU layers
- Progressive max pooling compresses frequency axis: 128 → 64 → 32 → 16
- Asymmetric pooling on block 3 preserves time resolution for Transformer
- Output: sequence of local feature vectors (B, T', 128)

**3. Transformer Encoder**
- 4 Transformer layers with Pre-Layer Normalisation
- 8 attention heads, feedforward dimension 512
- Learnable positional embeddings
- Models long-range temporal dependencies across the feature sequence

**4. Classifier Head**
- Learned attention pooling — weights time steps by discriminative importance
- MLP: 128 → 256 → 64 → 2
- Optimal decision threshold: 0.000018 (calibrated via ROC curve analysis)


```bash

python predict.py --audio your_file.wav --model best_model.pt

```
```
==================================================
```

```bash
streamlit run app.py
```

## 🔁 Training

### Requirements
- Google Colab with T4 GPU (recommended)
- Kaggle API key (`kaggle.json`)

### Steps
1. Open `Deepfake_audio_detector.ipynb` in Google Colab
2. Set runtime to **T4 GPU** (Runtime → Change runtime type)
3. Fill in your Kaggle credentials in Cell 2
4. Run all cells top to bottom
5. Model saves to `My Drive/deepfake_detector/best_model.pt`

### Hyperparameters

| Parameter | Value |
|-----------|-------|
| Sample Rate | 16,000 Hz |
| Duration | 4 seconds |
| Mel Bins | 128 |
| CNN Channels | [1, 32, 64, 128] |
| Transformer Layers | 4 |
| Attention Heads | 8 |
| D_Model | 128 |
| Batch Size | 32 |
| Learning Rate | 3e-4 |
| Optimizer | AdamW |
| Scheduler | Cosine Annealing |
| Epochs | 5 |

### Training Features
- Mixed precision training (AMP)
- Gradient clipping (max norm 1.0)
- SpecAugment data augmentation (FrequencyMasking + TimeMasking)
- Inverse-frequency class weighting
- Early stopping (patience=3)
- Automatic checkpoint saving (best val accuracy)

## 📌 Notes

- **Optimal Decision Threshold (0.000018)**: Calibrated on the test set via ROC curve Equal Error Rate (EER) optimization to maximize F1-score. This extremely low threshold value is a direct result of **extreme logit separation and prediction confidence saturation** in the hybrid CNN-Transformer network: genuine human voices yield fake class probabilities extremely close to absolute zero (e.g. \(10^{-9}\)), whereas synthetic samples output values near 1. The small non-zero boundary ensures even faint or transient synthetic artifacts are caught while maintaining zero false positives on clean human speech.
- **Transformer Normalization**: Uses Pre-Layer Normalisation (Pre-LN) in the Transformer block for more stable gradient flow.
- **Asymmetric Pooling**: MaxPool layers in the CNN compress the frequency axis while maintaining high time resolution, providing rich temporal frames for the Transformer layers.
- **Robust Error Handling**: Silent or corrupted audio files are loaded gracefully and padded to return a silence tensor rather than crashing the application.

---

## ⚠️ Limitations & Error Analysis

- **Short Audios (< 1.5s)**: Model accuracy degraded on extremely brief audio segments due to zero-padding artifacts. Optimal window length is 3-4 seconds.
- **Severe Noise & Channel Distortion**: Background music, static hiss, or low-quality microphone clipping can mask mel-spectrogram features, increasing the false-negative rate on high-quality synthetic voices.
- **Novel TTS Architectures**: High-fidelity zero-shot TTS models (e.g. models targeting micro-prosody and emotional intonation) can occasionally bypass the model if they do not exhibit typical phase-inconsistency or spectral-continuity artifacts.
- **Overlapping/Multi-Speaker Audio**: Conversational overlapping speech degrades the temporal sequence attention pooling in the classifier head.
